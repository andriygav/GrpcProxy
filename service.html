grpc_proxy.service.add_to_server(config, server)[source]¶

Generate proxy service handlers from the given config.

Parameters

  • config (dict) – Configuration dictionary from the setup.yaml file.

  • server – AInitialised server without handlers.

grpc_proxy.service.proxy_method(request, context, service, method, config)[source]¶

Prototype for gRPC proxy method.

Parameters

  • request (binary) – Binary request without deserialisation.

  • context – A gRPC service contex. For more details read https://grpc.github.io/grpc/python/grpc.html#service-side-context.

  • service (str) – A name of discovery service.

  • method (str) – A name of method in discovered service.

  • config (dict) – A config dictionary with all information. Must contain field ‘hosts’ and ‘loadBalancer’.

Returns

Responce from the target services.

Return type

binary
